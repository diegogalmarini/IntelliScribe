# Persuasion Principles for Skill Design

## Overview
Leverage psychological principles to ensure models follow critical workflows.

## The Seven Principles

### 1. Authority
Deference to expertise.
**Use:** "YOU MUST", "Never", "Always", "No exceptions".

### 2. Commitment
Consistency with prior actions.
**Use:** Require announcements ("I'm using [Skill Name]") and explicit choices.

### 3. Scarcity
Urgency from time limits.
**Use:** "IMMEDIATELY", "Before proceeding".

### 4. Social Proof
Conformity to norms.
**Use:** "Every time", "Always followed in this project".

### 5. Unity
Shared identity ("we-ness").
**Use:** "Our codebase", "We're colleagues".

### 6. Reciprocity
Obligation to return benefits. (Use sparingly).

### 7. Liking
Cooperativeness with those we like. (Avoid for compliance).

## Key Strategy
For **discipline-enforcing skills**, combine **Authority + Commitment + Social Proof**.
- "YOU MUST announce skill usage and watch the test fail first. This is how we ensure quality."
