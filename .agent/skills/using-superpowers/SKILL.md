# Using Superpowers

## Overview
Superpowers is an agentic framework for software development. It provides a set of composable skills to help you build, test, and debug code with high discipline.

## Skill Map

### Collaboration
- **brainstorming** - Turning ideas into designs and specs.
- **requesting-code-review** - Getting feedback on changes.
- **receiving-code-review** - Processing feedback.

### Development Methodology
- **using-git-worktrees** - Setting up clean workspaces.
- **writing-plans** - Planning implementation.
- **executing-plans** - Executing plans in batches.
- **test-driven-development** - Writing code via RED-GREEN-REFACTOR.
- **finishing-a-development-branch** - Integrating and cleaning up.
- **verification-before-completion** - Final quality checks.

### Problem Solving & Meta
- **systematic-debugging** - Root-cause analysis and defense-in-depth.
- **dispatching-parallel-agents** - Handling multiple independent tasks.
- **writing-skills** - Creating new skills.
- **subagent-driven-development** - Delegating to subagents.

## Core Philosophy
1. **Tests First** - Watch it fail before you fix it.
2. **Design First** - No implementation without an approved design.
3. **Trace to Source** - Fix root causes, not symptoms.
4. **Isolate Work** - Use worktrees and subagents.
5. **Verify proof** - Never trust, always verify.
